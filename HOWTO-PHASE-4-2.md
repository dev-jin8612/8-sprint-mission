# Phase 4-2: 애플리케이션의 기본 모니터링 설정

## 개요
Phase 4까지 완성된 CI/CD 파이프라인과 배포된 애플리케이션을 효과적으로 모니터링하기 위한 CloudWatch 설정을 구축한다. 프리티어 사용량 추적, 비용 관리, 성능 모니터링을 통해 안정적인 운영 환경을 구성한다.

---

## 1. CloudWatch 기본 설정

### 1-1. 로그 그룹 만들기
#### 1-1-1. 기존 로그 그룹 확인

**ECS 컨테이너 로그 그룹 확인:**
1. **AWS 콘솔** → **CloudWatch** 서비스로 이동
2. 좌측 사이드바에서 **로그** → **로그 그룹** 클릭
3. 기존에 생성된 로그 그룹 확인:
   - `/ecs/chap02-task-definition`: ECS 태스크의 애플리케이션 로그
   - `/aws/lambda/...`: Lambda 함수 로그 (있는 경우)
   - `/aws/rds/...`: RDS 로그 (있는 경우)

#### 1-1-2. 추가 로그 그룹 생성 (필요시)

**사용자 정의 로그 그룹 생성:**
1. **로그 그룹** 페이지에서 **로그 그룹 생성** 버튼 클릭
2. **로그 그룹 이름** 입력: `/custom/application-errors`
3. **보존 설정**: `1주일` 선택 (비용 절약을 위해)
4. **로그 그룹 생성** 버튼 클릭

> **💡 로그 보존 기간 설정 이유:**
> - **1주일**: 개발/테스트 환경, 디버깅용
> - **1개월**: 일반적인 애플리케이션 로그
> - **6개월 이상**: 규정 준수가 필요한 운영 환경
> - **Never expire**: 장기 보관이 필요한 중요 로그 (비용 주의)

### 1-2. 기본 지표 설정하기
#### 1-2-1. ECS 서비스 지표 확인

**ECS 관련 지표 탐색:**
1. **CloudWatch** → **지표** → **모든 지표** 클릭
2. **지표**에서 리전 확인 후 **ECS** 선택
3. **클러스터별**, **서비스별** 지표 확인:
   - **CPUUtilization**: CPU 사용률 (%)
   - **MemoryUtilization**: 메모리 사용률 (%)
   - **RunningTaskCount**: 실행 중인 태스크 수
   - **PendingTaskCount**: 대기 중인 태스크 수

#### 1-2-2. EC2 인스턴스 지표 확인

**EC2 관련 지표 탐색:**
1. **지표**에서 리전 확인 후 **EC2** 선택택
2. **인스턴스별** 지표 확인:
   - **CPUUtilization**: 인스턴스 CPU 사용률
   - **NetworkIn/NetworkOut**: 네트워크 트래픽
   - **DiskReadOps/DiskWriteOps**: 디스크 I/O 작업

#### 1-2-3. RDS 지표 확인

**RDS 관련 지표 탐색:**
1. **지표**에서 리전 확인 후 **RDS** 선택
2. **DBInstanceIdentifier** 지표 확인:
   - **CPUUtilization**: DB CPU 사용률
   - **DatabaseConnections**: 데이터베이스 연결 수
   - **ReadLatency/WriteLatency**: 읽기/쓰기 지연시간
   - **FreeStorageSpace**: 사용 가능한 저장 공간

### 1-3. 로그 보기
#### 1-3-1. 애플리케이션 로그 분석

**ECS 컨테이너 로그 확인:**
1. **로그 그룹** → `/ecs/chap02-task-definition` 클릭
2. **로그 스트림** 목록에서 최신 스트림 선택
3. **로그 이벤트** 분석:
   ```
   # 성공적인 애플리케이션 시작 로그 패턴
   2025-XX-XX INFO - Started Chap02AwsDeployApplication
   2025-XX-XX INFO - HikariPool-1 - Start completed
   2025-XX-XX INFO - Tomcat started on port 8080
   ```

#### 1-3-2. 로그 필터링 및 검색

**로그 검색 기능 활용:**
1. **로그 이벤트** 페이지에서 **필터 이벤트** 입력란 사용
2. **일반적인 검색 패턴**:
   - `ERROR`: 모든 에러 로그 검색
   - `"password authentication failed"`: 특정 에러 패턴 검색
   - `Started`: 애플리케이션 시작 로그만 표시
   - `Exception`: 예외 발생 로그 검색

3. **시간 범위 설정**:
   - 우측 상단 시간 선택기에서 **사용자 지정 범위** 설정
   - **지난 1시간**, **지난 24시간** 등 빠른 선택 활용

## 2. 알림 설정하기

### 2-1. 기본 알람 만들기
#### 2-1-1. CPU 사용률 알람 생성

**ECS 서비스 CPU 사용률 알람:**
1. **CloudWatch** → **알람** → **알람 생성** 클릭
2. **지표 선택** 단계:
   - **지표 선택** → **ECS** → **ClusterName, ServiceName**
   - `chap02-ecs-cluster`, `chap02-service` 선택
   - **CPUUtilization** 지표 선택
   - **지표 선택** 버튼 클릭

3. **조건 지정** 단계:
   - **임계값 유형**: `정적` 선택
   - **조건**: `보다 큼` 선택
   - **임계값**: `70` 입력 (70% CPU 사용률)
   - **다음** 버튼 클릭

#### 2-1-2. 메모리 사용률 알람 생성

**ECS 서비스 메모리 사용률 알람:**
1. 동일한 방법으로 **알람 생성** 시작
2. **지표 선택**: **MemoryUtilization** 선택
3. **조건 지정**: 임계값 `80` 설정 (80% 메모리 사용률)

#### 2-1-3. 프리티어 사용량 알람 생성

**EC2 인스턴스 실행 시간 알람:**
1. **지표 선택** → **사용량** → **EC2-Instance** → **Running-Hours**
2. **조건**: `보다 큼`, 임계값 `700` (월 750시간 중 700시간 초과 시 알림)

**RDS 인스턴스 실행 시간 알람:**
1. **지표 선택** → **사용량** → **RDS** → **InstanceTime**
2. **조건**: `보다 큼`, 임계값 `700`

### 2-2. 이메일 알림 설정하기
#### 2-2-1. SNS 주제 생성

**알림 주제 설정:**
1. **알람 생성** → **작업 구성** 단계에서:
2. **알람 상태 트리거** 선택:
   - **경보 상태**: 체크 ✅
   - **OK**: 체크 ✅ (알람 해제 시 알림)
   - **데이터 부족**: 체크 해제

3. **SNS 주제 선택**:
   - **새 주제 생성** 선택
   - **주제 이름**: `aws-monitoring-alerts` 입력
   - **알림을 받을 이메일 엔드포인트**: 본인 이메일 주소 입력
   - **주제 생성** 버튼 클릭

#### 2-2-2. 이메일 구독 확인

**구독 활성화:**
1. **생성된 SNS 주제로 이메일 발송**됨
2. **이메일 수신함 확인** (스팸 폴더도 확인)
3. 이메일 제목: `AWS Notification - Subscription Confirmation`
4. **Confirm subscription** 링크 클릭
5. **Subscription confirmed!** 페이지 확인

### 2-3. 알림 테스트하기
#### 2-3-1. 의도적인 부하 생성

**CPU 사용률 알람 테스트:**
```bash
# 현재 퍼블릭 IP 확인
curl http://{EC2_퍼블릭_IP}/actuator/health

# 동시 요청으로 CPU 부하 생성 (주의: 테스트 목적만)
for i in {1..50}; do
  curl http://{EC2_퍼블릭_IP}api/categories &
  curl http://{EC2_퍼블릭_IP}/api/menus &
done

# 잠시 대기 후 부하 중단
sleep 30
```

#### 2-3-2. 알람 동작 확인

**알람 상태 모니터링:**
1. **CloudWatch** → **알람** 페이지에서 생성한 알람 확인
2. **상태 변화 관찰**:
   - **OK** (정상) → **경보** (알람) → **OK** (복구)
3. **그래프 확인**: 지표 그래프에서 임계값 초과 지점 확인
4. **이메일 알림 수신 확인**:
   - 알람 발생 시: `ALARM: "알람이름" in Asia Pacific (Seoul)`
   - 알람 해제 시: `OK: "알람이름" in Asia Pacific (Seoul)`

## 3. 대시보드 만들기

### 3-1. 기본 대시보드 구성
#### 3-1-1. 대시보드 생성

**새 대시보드 만들기:**
1. **CloudWatch** → **대시보드** → **대시보드 생성** 클릭
2. **대시보드 이름**: `chap02-monitoring-dashboard` 입력
3. **대시보드 생성** 버튼 클릭

#### 3-1-2. 첫 번째 위젯 추가

**ECS CPU 사용률 위젯:**
1. **위젯 추가** → **행(선 그래프)** 선택
2. **지표 추가**:
   - **ECS** → **ClusterName, ServiceName**
   - `chap02-ecs-cluster`, `chap02-service` → **CPUUtilization** 선택
3. **위젯 옵션 설정**:
   - **제목**: `ECS Service CPU Usage`
   - **기간**: `5분`
   - **통계**: `평균`
4. **위젯 생성** 버튼 클릭

### 3-2. 주요 지표 추가하기
#### 3-2-1. 시스템 리소스 위젯

**메모리 사용률 위젯 추가:**
1. **위젯 추가** → **선 그래프** 선택
2. **ECS MemoryUtilization** 지표 선택
3. **제목**: `ECS Service Memory Usage`

**EC2 인스턴스 위젯 추가:**
1. **위젯 추가** → **선 그래프** 선택
2. **EC2 CPUUtilization** 지표 선택
3. **제목**: `EC2 Instance CPU Usage`

#### 3-2-2. 데이터베이스 위젯

**RDS 연결 수 위젯:**
1. **위젯 추가** → **선 그래프** 선택
2. **RDS** → **DatabaseConnections** 지표 선택
3. **제목**: `Database Connections`

**RDS CPU 사용률 위젯:**
1. **위젯 추가** → **선 그래프** 선택
2. **RDS** → **CPUUtilization** 지표 선택
3. **제목**: `Database CPU Usage`

#### 3-2-3. 실무에서 중요한 추가 지표

**네트워크 트래픽 위젯:**
1. **위젯 추가** → **선 그래프** 선택
2. **EC2** → **NetworkIn**, **NetworkOut** 지표 추가
3. **제목**: `Network Traffic`

**애플리케이션 응답 시간 위젯 (실무 추천):**
1. **위젯 추가** → **선 그래프** 선택
2. **ApplicationELB** → **TargetResponseTime** 지표 선택 (ALB 사용 시)
3. **제목**: `Application Response Time`

### 3-3. 대시보드 보기
#### 3-3-1. 대시보드 레이아웃 조정

**위젯 배치 최적화:**
1. **위젯 드래그**: 마우스로 위젯 위치 조정
2. **위젯 크기 조정**: 위젯 모서리 끌어서 크기 변경
3. **권장 레이아웃**:
   ```
   ┌─────────────┬─────────────┐
   │  ECS CPU    │  ECS Memory │
   ├─────────────┼─────────────┤
   │  EC2 CPU    │  Network    │
   ├─────────────┼─────────────┤
   │  DB CPU     │  DB Conn    │
   └─────────────┴─────────────┘
   ```

#### 3-3-2. 시간 범위 설정

**대시보드 시간 범위 조정:**
1. 우측 상단 **시간 범위 선택기** 클릭
2. **자동 새로고침 설정**:
   - **1분마다 새로고침** 선택 (실시간 모니터링)
   - **5분마다 새로고침** 선택 (일반적인 모니터링)

#### 3-3-3. 대시보드 저장 및 공유

**대시보드 저장:**
1. 우측 상단 **저장** 버튼 클릭
2. **대시보드 즐겨찾기 추가**: ⭐ 아이콘 클릭

## 4. 비용 모니터링 설정

### 4-1. 청구 알람 설정
#### 4-1-1. 청구 지표 활성화

**청구 알림 사전 설정:**
1. **AWS 콘솔** 우측 상단 **계정명** 클릭 → **계정** 선택
2. **빌링 기본 설정** → **알림 기본 설정** → **편집** 버튼 클릭 → 체크박스 활성화 후 **업데이트** 버튼 클릭

#### 4-1-2. 월 예산 초과 알람

**예상 청구액 경보 생성:**
1. **CloudWatch** (버지니아 북부) → **경보** → **경보 생성**
   > **⚠️ 중요**: 청구 지표는 `us-east-1` 리전에서만 사용 가능
2. **지표 선택**:
   - **Billing** → **예상 요금 합계** → **EstimatedCharges** 지표 선택
3. **조건 설정**:
   - **임계값**: `4` (월 4달러 초과 시 알림)
   - **알람 이름**: `monthly-billing-alert`
4. **SNS 알림**: 기존 생성한 이메일 주제 선택, 또는 새로 생성 후 선택

### 4-2. 서비스별 사용량 추적
#### 4-2-2. 주요 서비스 사용량 모니터링

**EC2 사용량 추적:**
- **지표**: `AWS/Usage` → `ResourceCount` → `EC2-Instance`
- **설명**: 실행 중인 EC2 인스턴스 수
- **프리티어 한도**: 월 750시간

**RDS 사용량 추적:**
- **지표**: `AWS/Usage` → `ResourceCount` → `RDS:db.t3.micro`
- **설명**: RDS 인스턴스 실행 시간
- **프리티어 한도**: 월 750시간

**S3 스토리지 사용량 추적:**
- **지표**: `AWS/S3` → `BucketSizeBytes`
- **설명**: S3 버킷 저장 용량
- **프리티어 한도**: 5GB

## 모니터링 베스트 프랙티스

### 실무에서 권장하는 지표 임계값
- **CPU 사용률**: 70% 이상 시 경고, 85% 이상 시 위험
- **메모리 사용률**: 80% 이상 시 경고, 90% 이상 시 위험
- **디스크 사용률**: 80% 이상 시 경고, 90% 이상 시 위험
- **응답 시간**: 500ms 이상 시 경고, 1초 이상 시 위험

### 알람 설정 원칙
- **False Positive 최소화**: 너무 민감한 임계값 설정 피하기
- **적절한 평가 기간**: 1-5분 간격으로 일시적 스파이크 무시
- **계층적 알림**: 경고 → 위험 → 심각 단계별 알림 설정
- **자동 복구**: 가능한 경우 Auto Scaling 등 자동 대응 방안 구축
